# Техзадание: MyKidTube — детское Android-приложение на базе YouTube API

> Версия 2 — обновлена по итогам уточнений (2026-04-08)

## 1. Обзор проекта

Нативное Android-приложение на Kotlin для планшета (Samsung SM-T819). Показывает контент из YouTube-аккаунта ребёнка: только видео с каналов, на которые он подписан. Цель — безопасная среда без рекомендательного алгоритма YouTube и нежелательного контента.

**Целевой пользователь:** ребёнок 4-5 лет.
**Тип проекта:** личный, публикация в Play Store не планируется (минимум на год).

---

## 2. Технический стек

- **Язык:** Kotlin
- **Min SDK:** Android 8.0 (API 26)
- **Целевое устройство:** Samsung SM-T819 (планшет)
- **Ориентация экрана:** Landscape (альбомная, фиксированная)
- **YouTube интеграция:** YouTube Data API v3 + YouTubePlayerView (`com.pierfrancescosoffritti.androidyoutubeplayer`)
- **Авторизация:** OAuth 2.0 через Google Sign-In SDK
- **Архитектура:** MVVM + Repository pattern
- **DI:** Hilt
- **Локальная БД:** Room (персистентное кэширование)
- **Сеть:** Retrofit 2 + OkHttp 3
- **Изображения:** Coil
- **Асинхронность:** Coroutines + Flow
- **Навигация:** Navigation Component + Bottom Navigation Bar (4 вкладки)
- **Тема:** Material Design 3, светлая/тёмная по системным настройкам

---

## 3. Авторизация

- При первом запуске — экран онбординга с кнопкой "Войти через Google"
- Открывается стандартный Google Sign-In (системный диалог / браузер)
- После успешного входа токен сохраняется локально (EncryptedSharedPreferences)
- При последующих запусках — автоматический старт на главном экране
- Автоматическое обновление токена в фоне (silent refresh)
- Смена аккаунта — только через настройки, защищённые родительским контролем

## 4. Экраны

### 4.1 Главный экран — Лента (Home)

- Сетка карточек видео (grid, 4 колонки для планшета)
- Контент формируется при каждом открытии/обновлении:
  1. Берётся полный список каналов подписки (до 50 каналов)
  2. Случайным образом выбираются **12 каналов**
  3. С каждого выбранного канала загружаются **последние видео** (первая страница, до 50 видео)
  4. Из видео каждого канала случайно выбираются 3-5 штук
  5. Итоговая лента перемешивается случайным образом
- Это позволяет контролировать влияние "активных" каналов на ленту — ни один канал не доминирует
- Каждая карточка: превью (thumbnail), название видео, название канала
- Pull-to-refresh — новая случайная выборка 12 каналов
- Пагинация / бесконечная прокрутка
- Кэширование ленты на 30 минут (Room, персистентное)
- **AppBar:** название приложения + иконка burger menu [☰] справа (вход в настройки)

### 4.2 История просмотров (History)

- Список видео, которые ребёнок открывал в приложении
- История хранится **локально** в Room (не из YouTube API)
- Сортировка: последние просмотренные сверху
- Каждый элемент: превью, название видео, дата просмотра ("Сегодня", "Вчера", ...)

### 4.3 Подписки (Subscriptions)

- Список всех каналов подписки
- Каждый элемент: аватар канала, название
- Кнопка "Отписаться" — **защищена родительским контролем**
- Нажатие на строку канала открывает список его видео

### 4.4 Поиск (Search)

- Поле ввода текста (debounce 800 мс)
- Поиск через YouTube Data API v3 (`search.list`) с параметром `safeSearch=strict`
- Результаты — только видео
- Карточки результатов в том же стиле, что и лента
- Информационная плашка: "🔒 Поиск фильтрует нежелательный контент"

## 5. Экран проигрывания видео (Player)

- Открывается при нажатии на любую карточку (полноэкранный Fragment)
- По умолчанию оверлей скрыт. При нажатии на экран появляется на 3 секунды, затем скрывается
- Оверлей:
  - **Верхняя часть (gradient):** кнопка "←" (слева), аватар канала + название видео + название канала (центр), кнопка "⋮" (справа)
  - **Нижняя часть (gradient):** прогресс-бар (красный — просмотрено, серый — остаток) + тайм-коды
- **Нажатие на аватар/название канала** → экран канала
- **Кнопка "⋮"** → меню "Заблокировать канал" (защищено родительским контролем)
- **Свайп вверх** → панель "Похожие из подписок" (видео продолжает играть):
  - 4 видео с того же канала (горизонтальный скролл)
  - 10 видео с других каналов из подписок (горизонтальный скролл)
  - Нажатие загружает новое видео в том же плеере
- **Back gesture заблокирован** — выход только через кнопку "←" в оверлее
- По окончании видео — плеер встаёт на паузу, появляется кнопка "Смотреть снова"
- Видео добавляется в локальную историю при открытии

## 6. Родительский контроль

При использовании защищённых функций отображается диалог с примером умножения двух чисел от 2 до 9. При правильном вводе ответа действие выполняется.

- **Запрашивается при:**
  - Попытке отписаться от канала
  - Блокировке канала через меню "⋮" в плеере
  - Входе в настройки через burger menu

**Надёжность:** ребёнку 4 года — таблица умножения ему недоступна. Пересмотр механизма — по достижении 7 лет.

## 8. Работа с YouTube Data API v3

| Действие | Метод API | Стоимость |
|---|---|---|
| Список подписок | `subscriptions.list?mine=true` | 1 ед. |
| Видео канала | `playlistItems.list` | 1 ед. |
| Детали каналов (batch) | `channels.list?id=...` | 1 ед. |
| Поиск | `search.list&safeSearch=strict` | 100 ед. |
| Отписаться | `subscriptions.delete` | 50 ед. |

**Квота:** 10 000 единиц/день.

**Расчёт дневного расхода:**
| Операция | Ед./раз | Раз/день | Итого |
|---|---|---|---|
| Обновление ленты (12 каналов) | ~14 | 10 | 140 |
| Поиск (родители) | 100 | 2 | 200 |
| Прочие запросы | — | — | ~50 |
| **Итого** | | | **~390** |

Квота не является ограничением.

**Стратегия кэширования:**
- Список подписок: 30 мин (Room)
- Лента: 30 мин (Room)
- Thumbnails: 24 ч (Coil disk cache)

**При превышении квоты:** full-screen сообщение "Лимит исчерпан. Попробуй завтра". Если кэш есть — показать кэш с баннером.

## 9. UI/UX требования

- Целевой пользователь: ребёнок 4-5 лет
- Крупные карточки и кнопки (min touch target 48dp), удобные для детских пальцев
- Минималистичный интерфейс без лишних элементов
- **Цветовая схема:** мягкие оттенки радуги (Material Design 3)
  - Primary: синий `#5B9BD5`
  - Secondary: оранжевый `#F4A261`
  - Tertiary: зелёный `#6CBF85`
- **Шрифт:** Nunito (или аналогичный округлый sans-serif)
- **Иконка приложения:** белый треугольник ▶ на синем фоне `#3A7DC9`
- Тёмный/светлый режим: следует системным настройкам (`MODE_NIGHT_FOLLOW_SYSTEM`)
- Skeleton-анимация для всех списков при загрузке
- Плавные анимации переходов (300 мс)
- **Пустые состояния:** иконка по центру + текст (без кнопок, если действие недоступно ребёнку)
- **Ошибка "нет сети":** иконка перечёркнутого Wi-Fi + кнопка "Повторить"
- **Прочие ошибки:** иконка предупреждения + кнопка "Повторить"

---

## 10. Что намеренно исключено

- Комментарии к видео
- Лайки / дизлайки
- Рекламные блоки (при использовании YouTubePlayerView поведение зависит от типа аккаунта — требует тестирования)
- Push-уведомления
- Многопользовательский режим
- Авто-воспроизведение следующего видео
- Offline-просмотр видео (только кэш метаданных)
