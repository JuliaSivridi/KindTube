# MyKidTube — UX-потоки и состояния экранов

> Версия 2 — обновлена по итогам ответов Product Owner (2026-04-08)

## Соглашения

- **Skeleton** — заглушка-анимация в форме контента (для списков)
- **Spinner** — круговой индикатор (для одиночных действий)
- **Empty state** — иконка по центру + текст
- **Error state** — специфическая иконка + текст + кнопка "Повторить"
- Целевой пользователь: ребёнок 4-5 лет. Все интерактивные элементы — крупные (min 48dp touch target)
- Ориентация: landscape, планшет Samsung SM-T819
- Тема: светлая / тёмная по системным настройкам

---

## Иконки состояний

| Состояние | Иконка Material | Описание |
|---|---|---|
| Пусто (история/лента) | `sentiment_satisfied` или `tv_off` | Нейтральная, дружелюбная |
| Пусто (поиск) | `search_off` | Нет результатов поиска |
| Нет сети | `wifi_off` (перечёркнутый Wi-Fi) | Понятно любому ребёнку |
| Ошибка (общая) | `error_outline` | Жёлтый треугольник с "!" |
| Квота API исчерпана | `hourglass_empty` | "Подожди до завтра" |

## 1. Онбординг / Авторизация

```
[Idle]
  Центр экрана:
    Иконка приложения (белый ▶ на синем фоне, 96dp)
    Название "MyKidTube" (крупный шрифт, primary color)
    Кнопка "Войти через Google" (крупная, заметная)

  ↓ нажатие кнопки
[Loading]
  Кнопка: spinner + "Входим..."
  Открывается Google Sign-In (системный диалог)

  ↓ успех → HomeFragment
  ↓ отмена/ошибка → Snackbar "Не удалось войти. Попробуй ещё раз" → [Idle]
```

## 2. Главный экран — Лента (Home)

### Структура экрана
```
┌─────────────────────────────────────────────────────────────────────┐
│  MyKidTube                                                          [☰] │  ← AppBar
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  [ Card ]  [ Card ]  [ Card ]  [ Card ]   ← Grid 4 колонки         │
│  [ Card ]  [ Card ]  [ Card ]  [ Card ]                             │
│  ...                                                                │
│                                                                     │
├─────────────────────────────────────────────────────────────────────┤
│  [🏠 Главная]  [🕐 История]  [📋 Подписки]  [🔍 Поиск]            │  ← Bottom Nav
└─────────────────────────────────────────────────────────────────────┘
```

**[☰] Burger menu** — правый верхний угол AppBar. Видна всегда. Нажатие → ParentalControl диалог → Настройки.

### Состояния

```
[Loading — первая загрузка]
  Grid: 8 skeleton-карточек (анимация shimmer)

[Success]
  Grid карточек, SwipeRefreshLayout сверху

[Refreshing — pull-to-refresh]
  Стандартный индикатор SwipeRefreshLayout
  Текущие карточки остаются видны
  ↓ успех → новая подборка (другие 12 каналов)
  ↓ ошибка сети → Snackbar "Нет подключения", оставить старую ленту

[Pagination — прокрутка вниз]
  Spinner в конце списка
  Подгружаются следующие карточки

[Empty — нет подписок]
  Центр экрана:
    Иконка tv_off (120dp, muted color)
    "Нет подписок"
    "Попроси родителей подписаться на каналы"
  Нет кнопок

[Error — нет сети, нет кэша]
  Центр экрана:
    Иконка wifi_off (120dp)
    "Нет подключения"
    Кнопка "Повторить"

[Error — квота API]
  Центр экрана:
    Иконка hourglass_empty (120dp)
    "Обновления временно недоступны"
    "Попробуй позже или спроси родителей"
  (Если кэш есть — показываем кэш + Snackbar с пояснением)
```

### Карточка видео
```
┌──────────────────────┐
│                      │
│      thumbnail       │  ← 16:9, Coil, placeholder — серый прямоугольник
│                      │
├──────────────────────┤
│ Название видео       │  ← 2 строки макс, TextAppearance.TitleSmall
│ Название канала      │  ← 1 строка, TextAppearance.BodySmall, onSurfaceVariant
└──────────────────────┘
```
Размер: карточки занимают ~23% ширины экрана (4 колонки с отступами 8dp).

## 7. Плеер (Player)

### Layout (реализовано)

Полноэкранный `PlayerScreen` (Composable). Три слоя в `Box`:

```
Layer 1 — AsyncImage (thumbnail)         ← фон пока iframe грузится
Layer 2 — AndroidView (WebView)          ← прозрачный, youtube-nocookie.com embed
Layer 3 — Compose Box (overlay)          ← наш UI, перехватывает все касания
```

### Текущий UI Layer 3

```
[Верхний оверлей] — видим 3 сек после касания, потом скрывается
  ┌────────────────────────────────────────────────────────────────────┐
  │ [←]  Название канала / Название видео (1 строка)         [🚫]     │
  └────────────────────────────────────────────────────────────────────┘

[Кнопка "Ещё видео ↑"] — видна только пока оверлей показан и панель скрыта
  Нажатие → открывает панель похожих видео

[Прогресс-бар] — всегда виден, красная линия 3dp у нижнего края
  Обновляется каждые 500 мс через webView.evaluateJavascript()

[Панель похожих видео] — выезжает снизу (~42% высоты)
  Горизонтальный LazyRow с видео (2-4 с текущего канала + остальные из подписок)
  Свайп вниз → закрыть
```

### Состояния
```
[Загрузка]
  Thumbnail виден через прозрачный WebView
  (YouTube встроенный spinner не показываем — controls=0)

[Играет — оверлей скрыт]
  Только видео + прогресс-бар внизу
  Касание → оверлей появляется на 3 сек

[Играет — оверлей виден]
  Верхняя панель + прогресс-бар + кнопка "Ещё видео"
  Через 3 сек без касания → оверлей скрывается

[Блокировка канала]
  Нажатие [🚫] → ParentalControl диалог
  Успех → канал в blocked_channels → переход на Home
  Отмена → продолжить смотреть
```

### Панель похожих видео
```
Видео продолжает играть.
Панель (~42% высоты, тёмный полупрозрачный фон):
  ┌──────────────────────────────────────────────────┐
  │  [Card] [Card] [Card] [Card] ...  ← LazyRow      │
  └──────────────────────────────────────────────────┘
Нажатие на карточку → новый PlayerScreen (popUpTo заменяет текущий, назад → исходный экран)
Свайп вниз → панель закрывается
```

## 8. Диалог родительского контроля

```
┌─────────────────────────────────────────────┐
│             Для родителей                   │
│                                             │
│      Сколько будет  7 × 8 ?                │
│                                             │
│  ┌─────────────────────────────────────┐   │
│  │           [  56  ]                  │   │  ← numberInput, крупный шрифт
│  └─────────────────────────────────────┘   │
│                                             │
│       [Отмена]        [Подтвердить]         │
└─────────────────────────────────────────────┘
```
- Числа: 2–9 (нет умножений на 0 или 1)
- Числовая клавиатура автоматически (`inputType = number`)
- Неправильный ответ: поле краснеет, пример заменяется новым
- Попытки не ограничены

## 10. Навигационные потоки

### Запуск приложения
```
App Start → проверка токена
    ├── Есть → HomeFragment
    └── Нет → OnboardingActivity → HomeFragment
```

### Из ленты/истории/поиска в плеер
```
[HomeFragment / HistoryFragment / SearchFragment]
  → нажатие на карточку
  → PlayerFragment (fullscreen, back stack)
  → [←] Назад → предыдущий экран (позиция прокрутки сохраняется)
```

### Из плеера в канал
```
PlayerFragment (оверлей виден)
  → нажатие на аватар/название канала
  → ChannelFragment
  → нажатие на видео → PlayerFragment (новый в back stack)
```

### Вход в настройки
```
HomeFragment → [☰]
  → ParentalControl диалог
      ├── Успех → SettingsFragment
      └── Отмена → остаться на HomeFragment
```

### Блокировка канала
```
PlayerFragment → [⋮] → "Заблокировать канал"
  → ParentalControl диалог
      ├── Успех → канал заблокирован → Snackbar → закрыть плеер → назад
      └── Отмена → продолжить смотреть
```

## 11. UX-принципы для 4-летнего ребёнка

| Принцип | Реализация |
|---|---|
| Крупные цели нажатия | Min 48dp, карточки и кнопки запас |
| Мало текста | Только название видео и канала на карточке |
| Предсказуемое поведение | Нажатие → видео. Всегда. Без сюрпризов |
| Нет случайных выходов | Back gesture заблокирован в плеере (только кнопка "Назад" в оверлее) |
| Понятные иконки | Только общеизвестные: ▶ ← ☰ |
| Мягкие анимации | Transitions 300ms, ease-in-out |
| Skeleton вместо spinner | Меньше тревожности при загрузке |
